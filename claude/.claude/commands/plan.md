# /plan Command

## Description
Create an implementation plan for a feature or task.

## Usage
```
/plan <feature description>
```

## Workflow
1. Analyze the requirement
2. Research existing code patterns
3. Identify files to modify/create
4. Outline implementation steps
5. Document testing strategy
6. List potential risks

## Output
Creates `implementation-plan.md` with:
- Approach summary
- Step-by-step implementation guide
- Files to modify/create
- Testing checklist
- Risk assessment

## Example
```
/plan Add user authentication with JWT
```
